<objective>
Turn the authenticated read-only workspace into the core labeling loop: one drawable/resizable/deletable bounding box per image with robust auto-save.
</objective>

<tasks>
<task type="auto" name="Add annotation persistence">
  <read_first>
  - .planning/phases/02-bounding-box-drawing-interface/02-CONTEXT.md
  - labeler/models.py
  </read_first>
  <action>
  Add an Annotation model keyed one-to-one to the image inventory record, normalized x/y/width/height fields, annotator tracking, timestamps, validation, admin registration, migrations, and authenticated JSON save/delete endpoints.
  </action>
  <acceptance_criteria>
  - Only one annotation row can exist per image.
  - Save endpoint records current user and timestamp.
  - Delete endpoint removes the current image annotation.
  </acceptance_criteria>
</task>

<task type="auto" name="Implement vanilla JS box overlay">
  <read_first>
  - website-mockup/app-screens.jsx
  - templates/labeler/workspace.html
  </read_first>
  <action>
  Add image overlay JavaScript for click-drag creation/replacement, move, four-corner resize, delete button, Delete/Backspace shortcut, min-size clamp, normalized coordinates, CSRF fetch helpers, and saved/error UI states.
  </action>
  <acceptance_criteria>
  - Dragging on empty image creates one box.
  - Dragging with an existing box replaces it.
  - Four corner handles resize without flipping inside-out.
  - Delete button and Delete/Backspace remove the box.
  </acceptance_criteria>
</task>

<task type="auto" name="Make auto-save resilient">
  <read_first>
  - .planning/phases/02-bounding-box-drawing-interface/02-CONTEXT.md
  </read_first>
  <action>
  Debounce saves after draw/move/resize/delete, retry transient failures, show a brief Saved confirmation after success, surface failure only after retries, and expose `window.nodulate.flushPendingSave()` for Phase 3 navigation.
  </action>
  <acceptance_criteria>
  - Saves are debounced during pointer movement.
  - `flushPendingSave()` resolves only after dirty state is saved or fails.
  - A visible Saved indicator appears after successful save.
  </acceptance_criteria>
</task>
</tasks>

<threat_model>
## STRIDE Notes

- **Tampering:** Mutation endpoints require authentication and validate normalized coordinate ranges.
- **CSRF:** JavaScript writes include Django CSRF token; no CSRF exemptions.
- **Information disclosure:** Responses return only annotation fields needed by the UI.
- **Repudiation:** Annotation rows store annotator and timestamps.
</threat_model>
