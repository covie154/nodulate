# Quick Task 260712-001: Env Superuser Bootstrap

## Goal

Allow an optional Django superuser to be created or promoted from `.env` values when the Docker container starts.

## Tasks

1. Add an idempotent management command that reads `DJANGO_SUPERUSER_USERNAME`, `DJANGO_SUPERUSER_PASSWORD`, optional `DJANGO_SUPERUSER_EMAIL`, and optional `DJANGO_SUPERUSER_UPDATE_PASSWORD`.
2. Run the command from the Docker entrypoint after migrations.
3. Document the env variables in examples and README without enabling placeholder credentials by default.
4. Add tests for skip, create, partial-config failure, existing user preservation, and explicit password rotation.

## Verification

- Run the Django test suite.
- Smoke-test the unset-env command path.
