# Quick Task 260710-001: Production Gunicorn/nginx Deployment

## Goal

Productionalise the Django app with Gunicorn, Docker Compose, HTTPS termination through nginx/certbot, env-file based configuration, S3-backed production image storage, and non-root/resource-capped containers.

## Tasks

1. Harden Django runtime configuration.
   - Require `DJANGO_SECRET_KEY` when debug is off.
   - Configure allowed hosts, CSRF origins, secure proxy headers, secure cookies, HSTS, static root, and SQLite path from env.
   - Add an unauthenticated health endpoint for container health checks.

2. Add production DICOM storage support.
   - Keep local filesystem discovery for testing.
   - Add S3 object listing and DICOM reads for production via boto3.
   - Keep generated PNG cache local to a writable container volume.

3. Add container deployment architecture.
   - Dockerfile runs the app as a non-root user.
   - Compose starts Gunicorn with three workers, nginx HTTPS termination, certbot init/renewal, volumes, and resource caps.
   - Env examples document local testing and EC2/S3 production settings without committing secrets.

4. Document operation.
   - README includes local container smoke test, first certificate issuance, production startup, and S3 configuration.

## Verification

- Run Django checks and tests with local settings.
- Render Compose config for testing env.
- Review git diff for secret leakage and deployment scope.
